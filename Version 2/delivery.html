The three.js comeback page

Location: site/index.html — single-file static page, no build step.

Tech: three.js r169 via ES module importmap, UnrealBloomPass for Volt glow, GPU particles for chalk-dust, scroll-driven camera + pulse animation.

4 scenes: resting pulse/hero → momentum fly-through → proof cards → Volt-green booking card.

Form: Name, email, phone (optional), class + time selectors, opt-in reminder checkbox. Submits to Formspree with mailto fallback.

Fallbacks: prefers-reduced-motion disables 3D scene, shows styled static layout. No-WebGL detection serves static hero + full form. <noscript> fallback with email contact.

Accessibility: aria-hidden canvas with real DOM content, keyboard-navigable, visible focus rings, AA contrast on all combos.

🐙 The Catch: Octopus caught lying

Bug: Missing async on initScene()

Who: Blue (Kojo), the three.js engineer.

What: initScene() was declared as a regular function but contained await expressions (await import('three')). In JavaScript, await is only valid inside async function. The page would throw a SyntaxError at runtime, breaking the entire three.js experience with a blank canvas.

Fix: Changed function initScene() → async function initScene() (applied during cold verify).

Why this matters: The no-WebGL fallback checks navigator... which returns false (WebGL is available) — so the fallback never fires. The page shows a blank canvas. The cold verifier caught what the dev missed.
